# Project Manifest: NRG Agent OS

## 1. Core Objective
NRG Agent OS is a structured, model-agnostic, local-first operating layer designed to govern and isolate distributed AI execution loops[cite: 2, 13, 15]. The fundamental goal is to make all agent activities completely repeatable, inspectable, and secure[cite: 5, 13, 15]. Under no circumstances are agents permitted to self-authorize system mutations, change policies, or freelanced directory operations[cite: 2, 13, 15].

## 2. System Paradigm
* **The OS is the Asset**: Large Language Models (LLMs) are treated as disposable computational engines[cite: 2, 13, 15]. The permanent lawbook of the system resides entirely in its contracts, interfaces, and transaction isolation structures[cite: 2, 13, 15].
* **In-Session Orchestration**: Execution loops run natively within a single master session utilizing cold-context subagents. External scripts or driver loops are entirely deprecated from core architecture.

## 3. Structural Layers
1. **Layer 1: Frontend Surface (HUD)** — Operational control surface displaying telemetry, system logs, active operating modes, and validation indicators[cite: 5, 13, 15].
2. **Layer 2: Backend OS API** — Strictly typed endpoints governing task intake, routing protocols, and filesystem security boundaries[cite: 5, 13, 15].
3. **Layer 3: Agent Runtime** — Execution sandbox managing model gateway adapters, cloud fallbacks, and local container resource pools[cite: 5, 13, 15].
4. **Layer 4: Memory Tier** — Strict data isolation layer splitting context across persistent, scratch, and isolated containment structures[cite: 5, 13, 15].
5. **Layer 5: Skill & Automation** — Modular registry of declarative, verified operational profiles[cite: 5, 13, 15].
6. **Layer 6: Validation & Safety** — Real-time automated runtime inspectors checking path, diff, credential, and behavioral parameters[cite: 5, 13, 15].
7. **Layer 7: Event / Ops** — Immutable, append-only systemic journal capturing every environment operation[cite: 5, 13, 15].
8. **Layer 8: Design System System** — Codified UI components communicating strict status, risk, and recovery parameters[cite: 5, 13, 15].
